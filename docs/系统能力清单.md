## 一、系统定位与边界

### 1.1 系统定位

面向职业院校的**数字课程服务与教学管理平台**，核心价值是：

- **课程生产**：教研管理员/教师构建体系化课程（体系课）与碎片化课程（颗粒课），配置学习资源、测验与评价规则
- **课程消费（学生端）**：学生浏览课程、按章节学习、完成测验、查看成绩与学习档案
- **课程消费（教师端）**：教师查看教学日程、管理课堂互动、分析学情数据
- **课程治理**：审批流程管理、批次分组管理、课程发布与版本控制

### 1.2 系统边界

| 边界项 | 说明/当前状态 |
|--------|-------------|
| 课程数据 | 本系统维护体系课与颗粒课的全生命周期数据 |
| 知识点数据 | 本系统维护知识点库，支持课程节点关联知识点 |
| 题库数据 | 本系统维护题库，支持选题组卷 |
| 外部系统集成 | 支持与门户首页、服务台、应用中心等外部平台切换 |
| 排课系统 | 教学日程为展示层，本期不做深度排课调度 |
| 直播教学 | 本期不做实时直播，视频为录播资源 |
| 多版本数据隔离 | 已发布课程更新版本后，学生学习进度重新关联（不做版本快照存储） |

## 三、核心实体与状态

### 3.1 课程（Course）

课程是平台的核心资源单元，分为**体系课**与**颗粒课**两种类型。

#### 基础信息字段

| 字段 | 说明 |
|------|------|
| 课程名称 | 必填 |
| 课程编码 | 系统自动生成，不可修改 |
| 课程类型 | 体系课（`system`）/ 颗粒课（`granular`） |
| 课程分类 | 公共基础课 / 数智化通识课 / 专业基础课 / 专业核心课 |
| 所属专业 | 单选，如软件工程、人工智能等 |
| 主讲教师 | 文本 |
| 所属行业 | 多选标签（电子信息、计算机行业、交通运输等 15 个行业） |
| 版本号 | 系统自动管理 |
| 更新日期 | 自动记录 |
| 节点数 | 体系课含节点数量统计 |
| 课时数 | 课程总课时统计 |
| 资源数 | 配套资源数量统计 |
| 浏览量 | 学生端浏览次数 |
| 学习人次 | 学生学习次数统计 |
| 课程封面 | 渐变色封面 |
| 课程标签 | 体系课 / 颗粒课 |

#### 课程状态流转

```
未提交(draft)
  ↓ 提交审批
审批中(pending)
  ↓ 审批通过          ↓ 审批驳回
已发布(published)    已驳回(rejected)
  ↓ 取消发布            ↓ 编辑 → 未提交(draft)
未提交(draft)
```

**各状态可执行操作**：

| 状态 | 可执行操作 |
|------|----------|
| 未提交(draft) | 编辑、删除、克隆、提交审批 |
| 审批中(pending) | 撤回审批（→draft）、删除、克隆 |
| 已驳回(rejected) | 编辑（→draft）、删除、克隆 |
| 已发布(published) | 取消发布（→draft）、克隆 |

**业务规则**：
- R-C01：任意状态均可克隆；克隆产生新课程，状态为未提交
- R-C02：已发布课程不可直接删除；需先取消发布
- R-C03：课程版本更新（重新发布新版本）后，学生学习进度清零，重新关联新版本
- R-C04：批次分组调整不触发重新审批，可随时调整

### 3.2 体系课节点（SystemCourseNode）

体系课由多层树形节点组成，支持父子层级嵌套。

#### 基础字段

| 字段 | 说明 |
|------|------|
| 节点名称 | 必填 |
| 节点顺序 | 数字，支持拖拽调整 |
| 节点类型 | 普通课程 / 颗粒课 / 克隆 / 引用 / 资源 |
| 父节点 | 可配置上级节点，形成树形结构 |
| 来源标识 | 克隆/引用类型时记录源课程 ID 与名称 |
| 教学目标 | 富文本，支持 Markdown 格式 |
| 建议时长 | 课时数（分钟） |
| 节点状态 | 与课程状态联动 |

#### 节点类型说明

| 类型 | 说明 |
|------|------|
| 普通课程（normal） | 在体系课中直接创建的节点，完全可编辑 |
| 颗粒课（original） | 引用已发布的颗粒课作为节点，内容同步 |
| 克隆（clone） | 基于颗粒课创建独立副本，可编辑 |
| 引用（quote） | 引用他人颗粒课，只读，不可编辑内容 |
| 资源（resource） | 纯资源节点，无教学内容配置 |

#### 节点内容配置

每个体系课节点包含以下配置模块：

| 模块 | 内容 |
|------|------|
| 教学目标 | 富文本编辑器输入，支持格式化 |
| 考查知识点 | 从知识点库中引用/克隆/新增；显示关联状态 |
| 配套资源 | 支持文档/表格/图片/链接/音频/视频等类型 |
| 课堂测验 | 从题库组卷，支持试卷（固定题目）和题库（随机抽题）两种模式 |
| 课后作业 | 文本要求描述，可配置是否需要附件提交 |
| 评价规则 | 针对已选测评形式配置评价点、评分主体、等级映射 |

### 3.3 颗粒课（Granular Course）

颗粒课是独立的小课程单元，可被体系课节点引用或克隆。

#### 编辑内容

| 模块 | 说明 |
|------|------|
| 基础信息 | 课程名称（自动生成编码）、课时数、学习目标、难度等级（1-5 星） |
| 知识点 | 从知识点库关联，支持多选 |
| 配套资源 | 资源上传与关联 |
| 测评形式 | 现场问答 / 现场评审 / 试卷 / 题库 |
| 评价规则 | 评价点配置、评分主体配置、等级映射、通过分数线 |
| 发布检查 | 发布前完整性检查面板 |

### 3.4 知识点（KnowledgePoint）

| 字段 | 说明 |
|------|------|
| 知识点名称 | 必填 |
| 编码 | 系统生成，可编辑 |

### 3.7 测评形式与评价规则

#### 四种测评形式

| 测评形式 | 说明 | 配置项 |
|---------|------|--------|
| 现场问答（random_draw） | 教师现场抽题提问 | 抽题数量、难度分布、评价点配置 |
| 现场评审（review） | 学生提交材料，教师评审 | 材料要求、评审步骤、评价量规 |
| 试卷（paper） | 固定试卷考核 | 试卷选择、考试时长、及格分数、重考次数、乱序、成绩展示、激活方式 |
| 题库（question_bank） | 从题库抽题组成测评 | 抽题数量、难度分布、选题列表、时间限制、重考次数 |

#### 评价点（EvalPoint）结构

| 字段 | 说明 |
|------|------|
| 评价点名称 | 必填 |
| 子类型 | 知识掌握 / 操作规范 / 任务完成等 |
| 评分方式 | 等级评分（level）/ 直接分数（score） |
| 等级映射 | 优秀(90-100)/良好(75-89)/及格(60-74)/不合格(0-59)，含颜色配置 |
| 权重 | 该评价点在总分中的占比 |
| 关联知识点 | 可关联已配置的知识点 |

#### 评分主体配置

| 评分主体 | 关键参数 |
|---------|---------|
| 教师 | 评分人数、多人记分规则（平均/中位数/最高/最低）、权重占比 |
| 企业导师 | 专业领域、最低工作年限、评分人数、权重占比 |
| 自评 | 是否需要反思、反思最低字数、权重占比 |
| 互评 | 互评人数、是否匿名、权重占比 |
| AI 评价 | AI 模型选择、权重占比 |
| 服务对象 | 服务方式、样本量、权重占比 |

**业务规则**：
- R-E01：各评分主体权重占比之和须等于 100%
- R-E02：测评对象（个人/小组）全局一致

### 3.8 批次分组（Batch）

用于对课程进行分组管理，不影响课程内容，不触发重新审批（R-C04）。

| 字段 | 说明 |
|------|------|
| 分组名称 | 必填 |
| 批次编号 | 系统自动生成，不可修改 |
| 关联审批流程 | 必填，关联一个审批流程模板 |
| 课程数 | 只读统计 |

---

### 3.9 审批流程模板（ApprovalWorkflow）

教研管理员预设的审批流模板，供批次关联使用。

| 字段 | 说明 |
|------|------|
| 流程名称 | 必填 |
| 流程说明 | 文本 |
| 审批步骤 | 动态列表，每步含：步骤名称/审批角色；至少保留 1 个步骤 |

**业务规则**：
- R-AP01：课程无批次时，提交审批时弹窗手动选择审批流程
- R-AP02：课程有批次时，使用批次关联的审批流程

### 4.2 审批流程

| 操作 | 触发角色 | 前置条件 | 产出/结果 |
|------|---------|---------|---------|
| 审核通过 | 有审批权限用户 | 状态=审批中 | 状态 → 已发布；可填写审批意见 |
| 审核驳回 | 有审批权限用户 | 状态=审批中 | 状态 → 已驳回；**驳回原因必填** |
| 发布课程 | 教研管理员 | 状态=审批通过 | 状态 → 已发布；学生端可见 |
| 取消发布 | 教研管理员 | 状态=已发布 | 状态 → 未提交；学生端不可见 |
| 管理审批流程模板 | 教研管理员 | — | 新增/编辑/删除审批流程模板 |
| 管理批次分组 | 教研管理员 | — | 新增/编辑/删除批次 |

**业务规则**：
- R-PUB01：已发布课程重新编辑须重走审批流程
- R-PUB02：发布新版本后相关学生学习进度清零
